# 블로그 후기 글 스타일 가이드

> 블로그 주인의 기존 맛집 후기 말투를 그대로 따라서 새 글을 작성한다.

## 말투 / 어미
- 친근하고 귀여운 존댓말: `~용`, `~어요`, `~답니다`, `~해용~~ㅎㅎ`
- 가끔 늘여쓰기: "다녀보고", "좋았어용", "나온답니다"
- 귀여운 감탄/마무리: `>ㅁ<`, `짠`, `빠잉`, `숭늉숭늉ㅎㅎ`, `ㅎㅎ`
- 이모지 적당히: 😊 ⏰ (남발하지 않기)

## 글 구조 (순서)
1. **인사** — 매번 다르게! (아래 "인삿말 규칙" 참고)
2. **오늘 소개할 곳** — 장소 이름 크게
3. **위치 / 접근성 / 분위기** — 골목, 역 거리, 조용한지 등
4. **영업시간 / 실용정보** — ⏰ 이모지와 함께 정리
5. **메뉴 소개** — 가격대, 뭘 시켰는지
6. **음식/사진별 코멘트** — 사진 하나하나에 짧고 귀여운 설명
   - 예: "따듯해서 기분좋아요", "이게 포인트", "짠 이런식으로 나와요"
7. **총평** — 좋았던 점 (조용함, 맛, 분위기 등)
8. **추천 멘트** — "데이트 코스로도 추천드려요!>ㅁ<"
9. **마무리 인사** — "빠잉"
10. **해시태그** — #지역맛집 #장소명 #메뉴 #분위기맛집 등 6~9개

## 인삿말 규칙 ⭐ (2026-07 사용자 피드백)
- **매번 똑같은 인사 금지!** "안녕하세요!! 다들 잘 지내셨나용~~ㅎㅎ" 복붙 X.
- **그날 계절·날씨·상황에 맞춰** 매번 다른 첫인사를 만든다. (귀여운 말투는 유지)
- 계절 키워드 예시:
  - 한여름(7~8월): "요즘 너무 덥죠 ㅠㅠ", "장마라 꿉꿉한데", "에어컨 없인 못 살겠어요", "시원한 거 땡기는 날씨"
  - 봄(3~5월): "벚꽃 피니까 설레는 요즘", "나들이하기 딱 좋은 날씨죠~"
  - 가을(9~11월): "선선해져서 살 것 같아요", "단풍 구경 다니기 좋은 계절"
  - 겨울(12~2월): "날이 너무 추워졌어요", "따뜻한 거 생각나는 계절이죠"
- 오늘 날짜(currentDate)를 기준으로 계절을 판단해서 자연스럽게 시작한다.
- "안녕하세요" 자체는 써도 되지만 뒤에 붙는 문장을 매번 새롭게.
- **첫인사("안녕하세요~")는 굵게/크게(.big) 강조하지 말 것.** 그냥 평범한 본문 크기로. (사용자 피드백)

## 폰트 크기 규칙 ⭐ (2026-07 사용자 피드백)
- 블로그 본문 글씨가 크다는 피드백(두 번). HTML `<style>`에서 아래 기준 사용:
  - `body` 본문: **13px**
  - `.big` (제일 큰 강조): **16px**
  - `.mid` (소제목): **15px**
- 안내 박스(.guide/.title)는 그대로 13.5~15px.

## 제목 규칙 ⭐ (검색 잘 되게 = 롱테일 SEO)
- **지역 + "맛집/카페" + 메뉴/특징 + 상호** 를 다 넣어 길게 쓴다.
- 지역은 **역 이름/동네**까지 구체적으로 (예: "건대입구", "신사역 가로수길").
- 예: `[건대입구 카페] 장원영 밀크티 맛집 차지티(CHAGEE) 건대점 웨이팅·앱주문 꿀팁`
- 예: `[신사역 가로수길 맛집] 분위기 좋은 항정살·파스타 다이닝 "한올" 데이트코스`
- 사람들이 실제로 검색할 법한 키워드(지역+메뉴+맛집)를 앞쪽에 배치.

## SEO·조회수 규칙 ⭐ (2026 네이버 상위노출 대응)
네이버 C-Rank(전문성) + D.I.A+(진짜 경험) 알고리즘 대응. 글마다 아래를 지킨다.
1. **사진 6장 이상** 넣기 (직접 촬영 원본). 5장 이하면 후보에서 더 고른다.
2. **롱테일 키워드 제목**: 유명 상호 단독보다 "지역+메뉴+특징"으로 구체화.
   (예: "송쉐프" ❌ → "남부터미널 만원 점심특선 깐쇼새우정식" ✅)
   본문에도 핵심 키워드를 자연스럽게 2~3회.
3. **진정성/솔직함**: 단점·개인 감상·에피소드를 꼭 한두 개 넣는다 (승부처).
4. **AI 티 줄이기**: 매번 같은 틀 복붙 금지. 문장·순서·표현을 조금씩 다르게, 개인 경험 한 스푼.
5. **체류시간용 분량**: 정보(주소·가격·팁)+후기로 충분히 길게 (너무 짧게 X).
6. **동영상 리마인드**: 발행 안내 때 "15초+ 짧은 영상(음식 클로즈업 등) 1개 넣으면 노출 유리"라고 사용자에게 알려준다.
7. **꾸준함(참고)**: 한 주제(맛집)로 꾸준히 쌓여야 C-Rank 전문성 인정 (카테고리당 50개+ 목표).

## 가게정보 규칙 ⭐ (2026-07 사용자 피드백)
- **"정확한 주소·위치는 네이버 지도에서 OOO 검색!" 같은 안내 문구는 넣지 말 것.** (사용자가 빼달라고 함)
- 가게 정보 섹션은 실제 주소·영업시간·전화 등만 적고, "검색해보세요" 류의 사족은 생략한다.

## 메뉴·가격 규칙 ⭐ (항상 포함)
- **모든 글에 "📋 메뉴/가격" 섹션을 반드시 넣는다.** (맛집·카페·핫도그 등 종류 무관)
- 사용자가 준 가격이 있으면 그대로, 없으면 WebSearch로 확인해서 넣되 "가격은 변동될 수 있어요" 문구 첨부.
- 가격을 못 찾으면 사용자에게 물어봐서라도 넣는다. (그냥 빼지 말 것)

## 사진 여러 장·그룹 규칙 ⭐ (2026-07 사용자 피드백)
- 네이버는 **사진을 한 구역에 여러 장 묶어(콜라주/그룹)** 올리는 기능이 있다.
- 그러니 **매장 내부/분위기 컷은 서로 다른 각도면 여러 장 넣어도 OK** (똑같은 각도 중복만 피하기).
- 사진 슬롯을 이렇게 묶어서 표시한다: `【 📷 사진 2,3,4_내부(여러 각도) 】`
  → 사용자가 그 자리에 3장을 한 그룹으로 드래그하면 됨.
- 음식도 접시 전체샷 + 클로즈업처럼 각도가 다르면 묶어서 여러 장 가능.
- 단, 거의 똑같은 사진(살짝 흔들린 같은 컷)은 1장만.

## 사진 방향(회전) 규칙 ⭐ (2026-07 사용자 피드백)
- **모든 사진을 올리기 전 방향을 반드시 확인한다.** (핸드폰 카메라 사진은 옆으로 눕는 경우 많음)
- 옆으로 누운 사진은 `sips -r 90/180/270`으로 회전 → **회전 후 그 파일을 Read로 다시 열어 정방향인지 눈으로 확인**한다. (한 번에 안 맞으면 각도 바꿔 재시도)
- 음식 클로즈업/위에서 내려다본 컷은 대체로 정방향이지만, 간판·외관·창밖·인물이 있는 컷은 특히 꼼꼼히 본다.
- 확신 안 서면 회전하지 말고 원본 유지 (잘못 돌리면 더 이상함). 최종 업로드 폴더 사진은 전부 정방향이어야 한다.

## 사진/내용 순서 규칙 ⭐ (2026-07 사용자 피드백)
- **상차림(먹는) 순서대로 배치**: 반찬 → 밑반찬 → 계란국/스프 같은 곁들이를 **앞쪽**에,
  대표 메뉴(메인 요리)는 **마지막**에 배치해서 클라이맥스로.
- 예: 반찬 → 계란국 → 사이드(만두/편육 등) → … → 메인 대표요리(맨 끝)

## 볼드체 규칙 ⭐ (2026-07 사용자 피드백)
- **일상 후기 문장(수다 부분)에는 볼드체(`<b>`) 쓰지 말 것.** 그냥 평범하게.
- 볼드는 아래처럼 **정보성/핵심**인 곳에만:
  - 메뉴·가격 섹션, 가게정보 섹션(영업시간·전화·주소 등 라벨)
  - 음식/음료 **이름** (예: 명이항정살구이, 백아절현)
  - "꿀팁!"·"포인트" 같은 라벨
- 문장 속 강조는 볼드 대신 **색상 span(.pink/.orange/.green 등)** 으로. (귀여운 포인트용)

## 톤 규칙
- 솔직하지만 따뜻하게. 과장 광고 느낌 X
- 문장 짧게, 줄바꿈 많이 (블로그 가독성)
- 사진 설명은 한두 줄로 귀엽게
- 정보(위치·시간·가격)는 정확하게 챙기기
